One-Time Payment Integrations

Choose the integration pattern that best fits your use case:

Recommended Integration

Standard, production-ready integration with PayPal, Pay Later, and PayPal Credit buttons. SDK automatically chooses the best presentation mode.

Best for: Standard checkout implementations

Server-Side Callbacks

Dynamic shipping calculation based on address with real-time tax calculation. Supports multiple shipping options and validates addresses.

Best for: E-commerce with shipping & tax calculations

Merchant Async Validation

Run custom validation logic before order creation. Shows loading screen while validation runs with configurable pass/fail scenarios.

Best for: Inventory checks, fraud prevention, account validation

Payment Handler Mode

Implements fallback strategy across presentation modes (payment-handler → popup → modal) for best UX across browsers.

Best for: Ensuring best UX across different browsers

Redirect Flow

Full page redirect to PayPal (like classic integrations). Uses hasReturned() and resume() to handle return from PayPal.

Best for: Mobile web, simple integrations, popup blocker issues

Direct App Switch

Mobile-optimized flow that switches to PayPal native app. Automatically opens PayPal app if installed, falls back to browser.

Best for: Mobile web where users have PayPal app

Save Payment Integration

Vault Payment Methods

Save payment methods for future transactions. Enables secure token storage and future payment processing for customer convenience.

Best for: Subscriptions, recurring payments, returning customers

Integration Tips

  • Start with Recommended: Try the recommended integration first to understand the basics
  • Server Callbacks: Essential for dynamic pricing, shipping, and tax calculations
  • Mobile Optimization: Use direct-app-switch or redirect for better mobile experience
  • Error Handling: Each demo includes comprehensive error handling patterns
